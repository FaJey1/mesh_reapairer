 # Архитектура mesh_reapairer

## Цели

- Обнаружение самопересечений сетки.
- Восстановление корректной геометрии сетки.
- Визуализация этапов восстановления.

## Чистая архитектура (в общих чертах)

- **domain**: описывает предметную область (структура сетки, алгоритмы поиска самопересечений, восстановления и базовая визуализация).
- **application**: orchestration-слой, который связывает доменные модули в сценарии использования (pipeline восстановления сетки).
- **infrastructure**: технические детали (загрузка/сохранение сеток, конкретные реализации визуализации на matplotlib и т.п.).
- **cli**: точка входа для командной строки, использующая application-слой.

## Основные доменные модули

- **msu**: описание структуры сетки (вершины, рёбра, грани, ячейки, атрибуты).
- **self_intersection_finder**:
  - `bvh_builder`: построение и обход BVH-дерева для ускорения поиска пересечений.
  - `intersection_finder`: поиск пересечений ячеек с использованием BVH.
  - `border_restorer`: восстановление корректной границы в области самопересечений.
  - `main`: композиция компонентов поиска самопересечений.
- **restorer**:
  - `restorer`: высокоуровневое восстановление сетки на основе найденных самопересечений.
  - `triangulator`: триангуляция ячеек/многоугольников.
  - `fixer`: удаление внутренней части самопересечения и финальная очистка.
- **vizualizator**:
  - `face_plotter`, `segment_plotter`, `line_plotter`, `plane_plotter`, `point_plotter`, `mesh_plotter` — специализированные функции/классы для визуализации разных сущностей.
